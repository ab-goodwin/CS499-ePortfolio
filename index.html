- Software Engineering and Database Design

Good software design is reliant on organization and separation of responsibilities. My approach focuses on making code modular, such as separating interface files from database operations, instead using dedicated helper class to simplify maintenance. In database design I prioritize data integrity and move towards unique primary keys. Expanding schemas to include structured fields to ensure data stays reliable and organized for long term business decisions.

- Algorithms and Data Structures

An application's performance depends how it handles data. By utilizing model classes and data structures such as ArrayLists, I can separate raw database tiers from the user interface. This lets me implment logic for searching, sorting, and filtering directly in memory before the display updates. This structured approach is quick and user friendly even as the dataset gets larger.

- Security

Security has to be build directly into an application's architecture from the beginning. Developing a security mindset means you need to anticipate how data can be compromised and use defensive coding to reduce those vulnerabilities early on. My technical apporach includes strict input validation, checking for empty fields, and hashing user sensative information. Protecting data upon input keeps the entire system safer and more reliable.

Summary of Technical Artifacts

The technical skills showcased in this portfolio highlights the incremental improvement of a mobile inventory management application. The project was a basic mobile app with tightly coupled logic and minimal features. I systematically imrpvoed it across three enhancements to demonstrate my skills in Computer Science:

Software Design and Engineering (Enhancement One):

Focused on structural organization and security improvements. I extracted the raw SQLite operations out of the activity files and centralized them into a new DatabaseHelper class. I also included password hashing and strict input validation to protect user information.

Algorithms and Data Structures (Enhancement Two):

Focused more on structure and performance. I moved away from relying soley on raw database cursors to display data, and implemented an InventoryItem model class, and ArrayLists to manage data in memory. This let me add dynamic searching, sorting, and filters.

Databases (Enhancement Three):

Focused on data integrity and database architecture. I changed the table structure to utilize a unique item ID as the primary key for all updates and deletions, replacing the difficult name based identifier. I also expanded the schema to include categories, and individual low stock threshholds.

Through these three enhancements, I demonstrated a cohesive and industry ready understanding of computer science principles, and turned a basic application into an optimized, secure, and more professional software solution.
